背景

最近需要在服务器上部署Mysql，出于业务需要选择了 Mysql 5.7.30，之前也有过安装 Mysql 的记录，一直没有记录过，装一次搜一次比较麻烦，这次将完整的安装步骤记录一下，便于以后查找。

下载

Mysql 官网提供了 Mysql 5.7.30 的下载链接，根据需要选择对应的服务器，这里选择 Red Hat Enterprise Linux 7。

Bundle 是下边所有包的集合，不想麻烦的话可以下载（除了安装包有些大..），最小化安装mysql 需要四个组件分别为：server, client, common, libs，这里选择最小化安装的方式。

安装

安装包下载完成后，通过 xftp 上传到服务器，进行安装，这四个组件的安装顺序为：

common -> libs -> client -> server, 如果顺序错了会提醒缺少依赖，按照提醒也能够找到正确的安装顺序。

安装 libs 时候提示了依赖冲突，与 mariadb-libs-5.5.44-2.el7 冲突了，这是因为 Red Hat 内置了 Mariadb 数据库替代原本的 Mysql（Oracle 收购了 Mysql 存在闭源风险），需要将这个包删除掉。通过：

登录

Github访问太慢

Posted on 2020-02-11 Edited on 2020-03-30 In Github
Symbols count in article: 1.6k Reading time ≈ 1 mins.

众所周知的原因，国内访问国外网站速度不是太快，甚至访问不了，作为最大同性交友网站Github 是众多程序员的圣地也是如此，同样难免带宽几百M，下载几十k的尴尬。提高pull/push代码的速度是众多程序员的最大诉求。

方法一（需要能科学上网）

  1. 查看科学上网端口 SS/SSR 默认局域网代理端口是: 1080，协议有两种http/socks5，本文sockets为例，http 操作类似。
  2. 设置全局代理

这种方式可以提高git clone https://www.github.com/xxx/xxx.git 的速度，但是不够完美，原因如下：

  • 挂了全局代理对国内的仓库会非常慢

  • 上述设置对 ssh协议 无效

针对全局代理对国内仓库访问会非常慢，可以只针对GitHub进行代理设置：

如果事先设置了全局代理，需要先取消：

针对 ssh 协议访问慢 需要修改 ~/.ssh/config 文件（不存在则新建, .ssh 为 windows 用户名下隐藏目录）

方法二（不需要科学上网）

git clone/push 走的不是 github.com 这个域名，而是 github.global.ssl.fastly.Net 这个域名，突破 github.global.ssl.fastly.Net 这个域名的限制可以提高 clone/ push 的速度，可以在hosts 中增加域名映射来绕过。

步骤：

  1. 访问 https://www.ipaddress.com 分别查到 github.com 和 github.global.ssl.fastly.Net 两个域名对应的 ip 地址
  2. 将上述映射记录添加到 hosts 文件中生效即可

ps: hosts 文件位置

windows( Administrator ) : C:\Windows\System32\drivers\etc\hosts

macOS / Linux ( root ) : /etc/hosts
